## §1. 서비스 정의

### 한 줄 정의

Daum은 검색·뉴스·커뮤니티·숏폼 영상을 하나의 앱에서 제공하는 한국의 포털 서비스이다.

### 핵심 사용 맥락

일상에서 반복적으로 열어 뉴스·트렌드·날씨 등을 확인하는 **습관형 정보 소비 앱**이다. 사용자는 특정 목적보다 "지금 이슈 뭐가 있나" 확인하는 패턴으로 앱을 이용하며, 세션당 평균 18~19스텝의 탐색 행동을 보인다.


### 플랫폼

- Android / iOS / PC 웹(daum.net) / 모바일 웹(m.daum.net)
- 앱의 경우에는 네이티브 영역으로만 구성된게 아니라 웹 영역을 프레임 안에 띄우는 페이지가 많음


### 핵심 수치

- 일평균 로그 수집량: 약 15억건
- 사용자 소비 행동의 98.4%가 3대 축(뉴스·홈탭·검색)에 집중
- 세션당 평균 스텝: 약 18~19스텝

### 한국 포털 시장 내 포지셔닝

- **네이버 대비**: 검색 점유율은 열세이나, 뉴스 소비·카페 커뮤니티에서 충성 사용자 보유
- **구글 대비**: 한국어 콘텐츠 생태계(카페, 블로그, 뉴스 제휴)에서 차별화
- **Daum만의 강점**: 뉴스 중심 포털로서의 정체성, 습관형 사용자 기반

**P1. 뉴스 재순환형 — "민수 씨" (41.1%)**
- 40~58세 직장인. 출퇴근 시간에 Daum 앱을 열어 뉴스를 확인하는 것이 습관
- 콘텐츠탭에서 기사 읽고 → 콘텐츠탭 복귀 → 다음 기사 선택을 세션당 2.6~2.9회 반복. **재순환이 본질**
- 체류시간에 따라 훑어보기형(콘텐츠탭 복귀 55~63%), 연속소비형(하단추천 클릭 15.9~23.4%), 완료후 종료형, 참여심화형으로 분화
- AI 니즈: 이슈 맥락 AI 요약, 관심사 기반 뉴스 큐레이션, 댓글 핵심 의견 요약, 재순환 추천 품질 향상
- Pain Point: 비슷한 기사 반복 노출, 원하는 깊이의 정보를 찾기 어려움
- AI 경험: ChatGPT 들어본 적 있으나 직접 사용은 거의 없음 → 기존 플로우에 자연스럽게 녹이는 접근 필수

**P2. 홈탭 탐색형 — "수진 씨" (37.3%)**
- 30~55세. 특별한 목적 없이 앱을 열어 홈 피드를 훑어보는 킬링타임 사용자
- 홈에서 피드 스크롤 → 채널/카페 들어갔다 → 다시 홈으로 돌아오기를 반복
- 체류시간별 의도 분화: 0~1.2초(즉시 전환 — 검색/종료), 1.2~6초(콘텐츠 선택 모드), 6~30초(피드 소비 진입 모드)
- AI 니즈: 의도 라우팅(체류·시간대·패턴 기반 맞춤 첫 화면), 개인화 피드, 홈→뉴스 경로 단축
- Pain Point: 홈에서 뭘 해야 할지 모르겠음, 관심 없는 콘텐츠가 섞여 있음, 10스텝에도 뉴스뷰 도달 44.7%
- 현재 "개인화 추천이 잘 동작하지 못함" → AI 개선의 체감 효과가 가장 큰 지점

**P3. 검색 목적형 — "현수 씨" (20.0%)**
- 전 연령대. 특정 정보를 찾기 위해 Daum 앱의 검색을 이용
- search→검색결과→검색컬렉션→검색결과 왕복 탐색 후 종료 또는 홈 복귀. 동선 응집도 88.8%
- AI 니즈: AI 검색 요약(즉시 답변), 왕복 해소, 한국 로컬 정보 특화 답변
- Pain Point: 원하는 답을 바로 찾기 어려움, 여러 번 왕복해야 함
- 2026년 6월 업스테이지 솔라 기반 AI 검색 탭 전환 예정 — 별도 프로젝트에서 주로 다룸

**P4. AI 네이티브 탐색자 — "지윤 씨" (현재 Daum에 부재)**
- 22~35세. ChatGPT·구글·유튜브가 정보 탐색의 첫 진입점. Daum은 "뭐하는 곳?" 수준의 인식
- AI 경험 높음(주 3회 이상, 평균 2.2개 서비스 병행). 기능적 우월성으로 서비스 선택
- Daum에 올 이유: 한국 로컬 뉴스·커뮤니티 데이터 특화 AI, 실시간 정보, 출처 인용
- H2~H3 단계의 장기 투자 타겟

**P5. AI 소극적 수용자 — "영숙 씨" (횡단)**
- 50~68세 중심이나 전 연령에 걸쳐 존재. AI에 대한 막연한 불안감 보유
- P1~P3의 하위 유형이 아닌, **모든 세그먼트를 가로지르는 횡단적 사용자 태도**
- AI 기능 설계 원칙: 점진적 노출, "AI" 라벨 최소화, 옵트인 방식, UI 안정성 유지

**우선순위**: P1·P2 동시 1순위(코어 수성) → P3 2순위(AI 검색 연계) → P4 장기 투자 / P5는 전 단계 관통

### 인증 및 개인화

- **인증**: 카카오 계정 통합 로그인. 비로그인 상태에서도 기본 서비스 이용 가능
- **개인화 요소**: 관심사 설정, 구독 언론사, 위치 기반 날씨, 주식/금융 위젯(코스피·HOT 종목), 추천 피드, 자주 가는 사이트(북마크)
- **사용자 권한**: 비로그인(읽기 전용) → 로그인(댓글·투표·구독·카페 활동) → 서비스별 등급(카페 등급 등)

## §4. 데이터 기반 핵심 문제 및 KPI

> 전체 분석 → `03_data_analysis.md` 참조
> 분석 기간: 2026.01.05~02.04 (4주), 총 9억건+ 로그, 100개 상태 정의

### 3대 핵심 문제

| 문제 | 설명 | 해결 방향 |
|------|------|----------|
| 홈 = 라우터인데 목적지처럼 다루고 있음 | 의도를 빠르게 연결 못함, 전이 분산 | 의도 라우팅 화면으로 재설계 |
| 뉴스 재순환 구조인데 재순환 품질 미관리 | 1회 읽기가 아닌 재순환이 본질 | 재탐색/연속소비 중심 운영 |
| 검색/숏폼/쇼핑을 홈과 동일 잣대로 봄 | 구조가 다른데 동일 평가 | 검색 독립 KPI, 숏폼·쇼핑 재검증 |

### 추적해야 할 핵심 KPI

**홈탭**: 첫 진입 후 첫 액션 분포, 재진입 후 종료 비율, 체류시간별 분기율, 2step/5step 뉴스뷰 도달률

**뉴스뷰**: 콘텐츠탭 복귀율, 하단추천 재클릭률, 즉시 종료율, 댓글 체류 후 참여행동 전환율, 세션당 뉴스뷰 수

**검색**: 결과↔컬렉션 왕복 횟수, 검색 시작→종료 전환 비율, 5-step exit 비율

## §5. 비즈니스 모델 및 정책

### 수익 모델

| 유형 | 설명 | 기타 | 
|------|------|------|
| DA (디스플레이 광고) | 홈탭 피드 중간중간에 광고 카드 삽입, 콘텐츠탭 상단 배너, 기사 상세 하단 등 주요 지면에 네이티브/배너 광고 노출. CPM 기반 | None | 
| SA (검색 광고) | 검색 결과 내 파워링크 영역. CPC 입찰 구조 | 쿼리에 따라 노출 여부 결정 |
| 커머스 연동 | 쇼핑탭, 톡딜 등 카카오 커머스 연동 수익 | 쇼핑으로 수익이 거의 발생하지 않고 조만간 없어질 예정 | 
| 스페셜 로고 광고 | 홈탭의 검색창 위에 큰 이미지로 광고를 게재 | 기념일에는 기념일 배너가 들어가는 자리

**광고와 이탈의 관계**: FEED_SLOT_AD_BIG_THUMB의 5스텝 이내 이탈이 46.4%, 혜택플러스 48.4% → 광고 경험이 세션 종료를 크게 가속. 광고 지면 설계 시 UX와 수익의 균형이 핵심 과제.

**광고 콘텐츠 식별**: 모든 광고 콘텐츠에는 "AD" 마크 또는 "ⓘ" 마크가 표시되어 일반 콘텐츠와 구분됨.

### 뉴스 편집 정책

- **편집 방식**: 알고리즘 편집 + 사람 편집 병행 (비중과 기준은 내부 정책)
- **제휴 구조**: CP(Content Provider)와 계약을 통해 뉴스 콘텐츠 수급
- **카테고리**: 콘텐츠탭 14개 서브탭(뉴스·언론사·라이브·스포츠·연예·쇼핑·머니·FUN·홈&쿠킹·스타일·여행맛집·지식교양·자동차+·동물)
- **규제**: 포털 뉴스 편집 관련 규제 동향 지속 모니터링 필요

### 댓글(타임톡) 정책

- **세이프봇**: AI 기반 콘텐츠 모더레이션. 커뮤니티 투표·기사 댓글에서 "세이프봇 작동 중" 안내 노출
- **댓글 구조**: 닉네임 + 시간 + 본문 + 답글 + 감정 이모지 필터 + 정렬(최신순 기본)
- **참여 현황**: 참여/댓글형 클러스터 방문율 11.4%이나 행동 밀도 높음

## §6. 경쟁 환경 요약

### 한국 포털 점유율 구도

**검색 시장**: 네이버 > 구글 > Daum (Daum은 3위이나 격차 큼)
**뉴스 소비**: 네이버 뉴스 > Daum 뉴스 (Daum의 뉴스 충성도는 높은 편)
**커뮤니티**: 네이버 카페 > Daum 카페 (Daum 카페는 특정 영역에서 강세)

### 영역별 주요 경쟁자

| 영역 | 경쟁자 | Daum 대비 특성 |
|------|--------|--------------|
| 검색 | 네이버, 구글 | 네이버는 자체 콘텐츠 생태계 강점, 구글은 글로벌 AI 기술 우위 |
| 뉴스 | 네이버 뉴스, 구글 뉴스 | 네이버는 뉴스 편집 체계 선도, 구글은 개인화 추천 |
| 커뮤니티 | 네이버 카페, 당근마켓, 에브리타임 | 당근은 로컬, 에브리타임은 대학생 버티컬로 분화 |
| 숏폼 | 틱톡, 유튜브 쇼츠, 인스타 릴스 | 전문 플랫폼 대비 Daum 루프의 콘텐츠 경쟁력 부족 |
| 쇼핑 | 네이버 쇼핑, 쿠팡 | 전문 커머스 대비 Daum 쇼핑의 목적성 부족 |

### SWOT 요약

| | 내용 |
|-|------|
| **Strengths** |  뉴스/카페 충성 사용자, 3대 축 98.4% 집중(명확한 코어), 일 15억건 로그 자산 |
| **Weaknesses** | 검색 점유율 하락, 쇼핑/숏폼 흡수 실패(합계 1.6%), 신규 사용자 유입 부족, 로그 데이터 활용 미흡 |
| **Opportunities** | AI 전환("나를 가장 잘 아는 뉴스 미디어"), 데이터 기반 개인화 고도화, 선행지표 중심 운영 전환 |
| **Threats** | 구글 AI 검색 확대, SNS 기반 정보 소비 전환, 포털 뉴스 규제 강화, 버티컬 플랫폼의 포털 기능 대체 |

### 최근 산업 트렌드 키워드

- **AI 검색**: Perplexity, Google SGE 등 답변형 검색의 부상
- **숏폼 콘텐츠**: 정보 소비 패턴이 텍스트에서 영상으로 이동
- **버티컬 플랫폼**: 당근(로컬), 토스(금융), 쿠팡(쇼핑) 등이 포털 기능을 대체
- **뉴스 소비 변화**: SNS 기반 뉴스 소비 증가, 포털 뉴스 의존도 감소 추세
