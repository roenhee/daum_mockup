# Daum 앱 화면 스펙

> **목적**: AI가 Daum 앱의 화면별 구조, UI 요소, 인터랙션을 정확히 파악하여 와이어프레임 설계·PRD 작성·UI 기획을 지원할 수 있도록 한다.
> **범위**: 모바일 앱 (Android/iOS) 기준. PC 웹은 이 문서에서 다루지 않음.
> **참조**: 서비스 맥락·사용자·데이터 → `01_daum_context.md` / 정량 분석 → `03_data_analysis.md`
> **최종 수정일**: 2026-03-23
> **작성자**: Roen.hee

## 1. 글로벌 구조

### 공통 UI 요소

- **상단 헤더**: 탭마다 구성이 다름 (아래 탭별 상세 참조). 검색 아이콘은 대부분의 탭에서 공통 제공.
- **검색바**: 홈 탭에서는 독립 영역으로 노출, 스크롤 시 헤더로 축소·병합. 콘텐츠 탭에서는 아이콘 형태.
- **하단 탭 바**: 현재 5개 탭 고정 (4분기 6탭 예정). 활성 탭은 컬러 아이콘으로 구분.
- **하단 고정 영역**: 탭 화면에서는 하단 탭 바가, 상세 뷰 화면(뉴스 기사 상세·채널뷰·검색 결과 등)에서는 인앱 브라우저 툴바(Daum 로고 + ← → 네비게이션 + 검색 + 공유 + 더보기)가 모바일 화면 하단에 항상 고정됨.

### 구현 방식 구분

앱 전체가 네이티브로 구성된 것이 아니라, 웹 영역을 프레임 안에 띄우는 페이지가 많음.

**네이티브 영역**: 홈탭 전체, 사이드 메뉴, 각 탭의 상단(타이틀·서브탭 바), 하단 탭 바
**웹뷰 영역**: 홈탭·사이드 메뉴를 제외한 나머지 탭의 본문 콘텐츠 영역 전체

| 영역 | 구현 방식 | 비고 |
|------|----------|------|
| 홈탭 | **전체 네이티브** | 피드, 위젯, 검색바 등 모두 네이티브 |
| 사이드 메뉴 | **전체 네이티브** | 햄버거 메뉴 |
| 하단 탭 바 | 네이티브 | 전 화면 공통 |
| 콘텐츠탭 — 타이틀·서브탭 바 | 네이티브 | 상단 셸만 네이티브 |
| 콘텐츠탭 — 본문 콘텐츠 | 인앱 브라우저 | 뉴스 리스트·피드 등 |
| 커뮤니티탭 — 타이틀·서브탭 바 | 네이티브 | 상단 셸만 네이티브 |
| 커뮤니티탭 — 본문 콘텐츠 | 인앱 브라우저 | 투표·카페 인기글 / 개편 후 소셜 피드 |
| 쇼핑탭 — 타이틀·서브탭 바 | 네이티브 | 상단 셸만 네이티브 |
| 쇼핑탭 — 본문 콘텐츠 | 인앱 브라우저 | 커머스 피드 |
| 루프탭 — 타이틀·서브탭 바 | 네이티브 | 상단 셸만 네이티브 |
| 루프탭 — 본문 콘텐츠 | 인앱 브라우저 | 숏폼 영상 피드 |
| 뉴스 기사 상세 | 인앱 브라우저 (v.daum.net) | 기사 본문·타임톡·하단 추천 |
| 채널뷰 | 인앱 브라우저 (v.daum.net) | 채널 게시글 상세. 뉴스 기사 상세와 별도 페이지 |
| 검색 결과 | 인앱 브라우저 (m.search.daum.net) | 통합검색·버티컬 검색 |

### 사이드 메뉴 (햄버거 메뉴)

- **진입**: 홈 탭 좌측 상단 햄버거(≡) 아이콘
- **구성**:
  - 프로필 영역: 사용자명 + 이메일 + 프로필 편집
  - 메일/카페 바로가기 탭
  - 서비스 바로가기 그리드: 금융, 뉴스, 스포츠, 날씨, 카카오맵, 다음사전, 웹툰, 전체서비스
  - 자주 가는 사이트: 개인화된 북마크 리스트
  - 공지사항 배너
  - 하단: 글꼴 설정(가+), 로그아웃

## 2. 탭별 화면 상세

### 2-1. 홈 탭

> **구현**: 전체 네이티브
> **헤더**: 햄버거 메뉴(좌) + 프로필·AI·알림 아이콘(우)
> **스크롤**: 세로 무한 스크롤
> **참조 이미지**: `screenshots/home_tab.png`

**① 실시간 트렌드** — 순위형 키워드 리스트. 기본 2건 노출, 펼치면 10건 노출. 펼치기/접기 토글. 각 항목에 신규(N)·상승(▲) 뱃지.

**② 위젯 캐러셀** — 가로 스크롤 카드. 날씨(위치 기반 기온·특보), 주식(코스피 지수·등락), 스포츠(경기 일정·스코어), 커뮤니티(인기글 미리보기) 등 개인화 위젯.

**③ 이슈 슬롯** — 썸네일 없는 텍스트 전용 리스트 4건. 한 줄 말줄임 처리.

**④ 뉴스 피드 카드** — 제목(2줄) + 우측 썸네일 + 출처·시간·더보기(···) 구조의 카드 반복. 공식 언론사 기사와 채널 게시글이 혼재되어 노출됨. 기사 탭 시 뉴스 기사 상세(§3-1)로, 채널 게시글 탭 시 채널뷰(§3-2)로 이동.

**⑤ 구독한 언론사** — 가로 스크롤 언론사 아이콘 리스트 + 해당 언론사의 대형 기사 카드(전면 이미지 + 제목 오버레이).

**⑥ 실시간 라이브** — 영상 썸네일 + LIVE 뱃지 + 시청자 수. 하단에 "라이브 더보기" CTA.

**⑦ HOT 종목 현황** — 종목별 주가 정보 위젯. 종목명 + 현재가 + 등락률. 여러 종목 리스트.

**⑧ 추가 뉴스 피드** — ④와 동일한 카드 패턴이 무한 스크롤로 반복. 기사·채널 게시글·쇼핑 콘텐츠 등이 혼재. 피드 중간중간에 광고 카드가 삽입되며, 광고 카드에는 "AD" 마크 또는 "ⓘ" 마크가 표시됨.

**⑨ 실시간 트렌드 (하단)** — 피드 하단부에 실시간 트렌드가 다시 한번 노출.

**홈 탭 UI 패턴**:
- 검색바가 스크롤 시 헤더로 축소·병합되는 트랜지션
- 가로 스크롤: 위젯 캐러셀(②)과 구독 언론사 아이콘(⑤)
- 회색 구분선으로 주요 섹션 분리
- 개인화: 날씨(위치 기반), 주식(코스피), 구독 언론사, 추천 피드
- 콘텐츠 혼재: 기사·채널 게시글·쇼핑 콘텐츠·금융 위젯이 하나의 피드에 섞여 노출

### 2-2. 콘텐츠 탭

> **구현**: 타이틀·서브탭 바 = 네이티브 / 본문 콘텐츠 = 인앱 브라우저
> **헤더**: "콘텐츠" 타이틀 + 설정(필터) 아이콘 + 검색 아이콘
> **스크롤**: 세로 무한 스크롤
> **참조 이미지**: `screenshots/contents_news.png`

**서브탭 바 (가로 스크롤)**: 뉴스(기본 활성) · 언론사 · 라이브 · 스포츠 · 연예 · 쇼핑 · 머니 · FUN · 홈&쿠킹 · 스타일 · 여행맛집 · 지식교양 · 자동차+ · 동물 — 총 14개. 활성 탭에 밑줄 인디케이터.

#### 뉴스 서브탭 (기본 활성)

**① 광고 배너** — 상단 광고 영역. "AD" 마크 또는 "ⓘ" 마크 표시.

**② 주요 뉴스 리스트** — 텍스트 전용 헤드라인 리스트(5건). 출처 + 시간 표시. 하단에 "새로운 주요 뉴스 1/8" 페이지네이션 + 필터 아이콘.

**③ 날씨 위젯** — 현재 기온 + 날씨 상태 + 위치. 홈 탭 위젯보다 간소한 한 줄 형태.

**④ 공지/안내 배너** — 유사시 알림 안내 등 공지성 배너가 노출될 수 있음.

**⑤ 이 시각 추천뉴스** — 제목(1줄) + 출처·시간 구조의 텍스트 리스트. 무한 스크롤.

**⑥ 포토 뉴스 캐러셀** — 가로 스크롤. 정사각 썸네일 + 제목(2줄) 카드.

**⑦ 금융 위젯** — 카카오뱅크·카카오증권 등 금융 서비스 위젯. 주가, 금융 정보 등. 전체 사용자 대상 노출 (개인화 아님).

**⑧ 추가 콘텐츠 피드** — 뉴스 기사·채널 게시글·연예 뉴스·언론사별 뉴스 등이 혼재되어 무한 스크롤. 섹션별로 "연예뉴스", "언론사 뉴스" 등 헤더로 구분.

**⑨ LIVE 뉴스** — 실시간 방송 중인 뉴스 영상 (YTN 등). 썸네일 + LIVE 뱃지 + 제목.

#### 언론사 서브탭

- 언론사별 기사 리스트 + 구독/구독취소 버튼

#### 라이브 서브탭

- 현재 진행 중인 라이브 영상 리스트
- 영상 썸네일 + LIVE 뱃지 + 시청자 수 + 제목
- 예정된 라이브 일정 노출

#### 스포츠 서브탭

- 종목별 서브 필터 (야구·축구·농구·배구·골프 등)
- 경기 일정·스코어 카드 (팀 로고 + 점수 + 상태)
- 스포츠 뉴스 피드 (썸네일+제목 카드, 뉴스탭과 동일 패턴)
- 순위표/기록 링크

#### 연예 서브탭

- 연예 뉴스 피드 (썸네일+제목 카드)
- 포토 뉴스 캐러셀 (대형 이미지 중심)
- 인기 기사 랭킹 리스트

#### 기타 서브탭 공통 패턴 (쇼핑·머니·FUN·홈&쿠킹·스타일·여행맛집·지식교양·자동차+·동물)

- 대부분 뉴스탭과 유사한 구조: 카테고리 특화 뉴스/콘텐츠 피드
- 썸네일+제목 카드 + 출처·시간 형태의 무한 스크롤
- 일부 탭은 상단에 카테고리 특화 위젯 포함 (예: 쇼핑→특가 배너, 머니→시세 위젯)
- 각 서브탭마다 콘텐츠 소싱이 다름 (뉴스 CP 기사 / 채널 게시글 / 블로그·카페 UGC / 제휴 콘텐츠 혼합)
- 일부 서브탭에서는 공식 언론사 기사와 채널 게시글이 혼재되어 노출됨

### 2-3. 커뮤니티 탭 (2026년 4월 말 전면 개편 예정)

> **현재 구현**: 타이틀·서브탭 바 = 네이티브 / 본문 콘텐츠 = 인앱 브라우저
> **현재 헤더**: "커뮤니티" 타이틀 + 검색 아이콘
> **스크롤**: 세로 무한 스크롤
> **참조 이미지**: `screenshots/community_sandbox.png` (개편 후)

#### 현재 구조 (개편 전)

**① 이슈를 말하다 (투표/토론)** — 섹션 헤더 + "세이프봇 작동 중" 안내. 카드 구조: 출처(언론사) + 투표 참여자 수 + 토론 주제(제목+설명) + 선택지(라디오 버튼) + "투표하기"/"결과보기" 버튼. 여러 투표 카드가 세로로 반복.

**② 지금 N시 카페 인기글** — 시간대별 인기 카페 글 순위 리스트. 순위 번호 + 제목(2줄) + 카페명 + 우측 썸네일 구조. 5건 이상 노출.

#### 변경 후 구조 (개발 중 — 샌드박스 기준)

쓰레드(Threads)와 유사한 형태의 사용자 게시글 기반 커뮤니티. 뉴스·이슈에 대한 사용자 의견과 정보를 공유하는 소셜 피드.

> **헤더**: 알림(벨) 아이콘 + 검색 아이콘
> **하단 FAB**: "글쓰기" 버튼 (펜 아이콘 + 텍스트, 우하단 고정)

**서브탭 바**: 실시간(기본 활성) · 발견 — 2개 탭. 활성 탭에 굵은 글씨 + 밑줄 인디케이터.

**실시간 서브탭**:

**① 인기 토픽** — 시간 기준 인기 키워드 순위 리스트. "인기 토픽" 헤더 + 기준 시간(예: "2026.03.23 17:00:00 기준") + 정보(ⓘ) 아이콘 + 접기/펼치기(∨). 2열 구성으로 01~06 순위 표시. 각 항목은 둥근 칩(pill) 형태.

**② 추천글** — "추천글" 섹션 헤더. 개인화 기반 추천 게시글 피드.

**③ 게시글 피드 (무한 스크롤)** — 사용자 게시글이 세로로 반복. 카드 구조:
- 프로필: 사용자 아이콘 + 이름 + 팔로우(+) 버튼 + 카테고리 뱃지(예: "경제") + 작성 시간(예: "5분 전")
- 본문: 텍스트 + 해시태그형 토픽 뱃지(예: "임차인조심", "휴림로봇 주식손실") + 이미지/동영상 임베드 + 외부 링크 프리뷰(Daum 기사 등) + 주식 차트 등 리치 콘텐츠
- "AI 자동완성" 버튼 — 게시글 작성 시 AI 기반 텍스트 자동완성 기능
- 인터랙션 바: 좋아요(아이콘+수) + 댓글(아이콘+수) + 공유(종이비행기 아이콘) + 더보기(···, 우측 끝)
- 카드 간 회색 구분선

**④ 나와 비슷한 사용자 추천** — 사용자 추천 카드. 프로필 아이콘 + 이름 + "팔로우" 버튼. 관심사 기반 팔로우 추천.

**⑤ 떠오르는 핫 이슈** — 현재 화제인 이슈 관련 게시글 모음. 영상/이미지 포함 게시글 카드.

**발견 서브탭**:

**① 오늘의 추천 토픽** — 2×2 대형 이미지 그리드. 각 카드는 배경 이미지 + 토픽 제목 오버레이. 탭하면 해당 토픽 관련 글 모아보기.

**② 지금 가장 인기 있는 토픽** — 가로 스크롤 칩 리스트. 첫 번째 칩은 강조색(보라) 배경. 탭하면 해당 토픽 관련 대표 게시글로 이동.

**③ 토픽별 인기 게시글** — 선택된 토픽의 대표 게시글 카드. 실시간 서브탭의 게시글 카드와 동일한 구조. 무한 스크롤.

## 3. 주요 세부 화면

> Daum의 콘텐츠는 크게 두 가지 유형이 있으며, 각각 별도의 상세 화면을 가진다.
> - **기사**: 공식 제휴 언론사(CP)가 생산하는 뉴스 콘텐츠 → **뉴스 기사 상세** (§3-1)
> - **채널 게시글**: 일반 사용자·크리에이터가 준 공식 언론처럼 작성·게시하는 콘텐츠 → **채널뷰** (§3-2)
>
> 기사와 채널 게시글은 별도 탭에 분리되어 있지 않고, 홈탭 피드·콘텐츠탭 일부 서브탭에서 혼재되어 노출됨.

### 3-1. 뉴스 기사 상세

> **구현**: 인앱 브라우저 (웹뷰, v.daum.net)
> **진입**: 홈·콘텐츠 등에서 뉴스 기사 카드 탭
> **참조 이미지**: `screenshots/news_detail.png`

**상단 바**: ← 뒤로가기 + URL 표시(v.daum.net) + 새로고침 + 탭 카운트

**언론사 헤더**: Daum 로고 + 언론사명 + 구독 버튼 + 언론사 홈·검색 아이콘

**기사 본문 영역**:
- 제목(대형 텍스트) + 기자명 · 작성일시
- 조회수 + 유틸리티 아이콘(필터·음성읽기·AI요약·글꼴크기)
- 본문 텍스트 + 인라인 이미지 + 날짜 스탬프
- 관련 영상 (인라인 비디오 플레이어, 해상도 선택: 360/480/720/1080)
- 본문 하단: 기자 정보 + 저작권 안내

**기사 하단 인터랙션**:
- 감정 반응 투표: 추천해요·좋아요·감동이에요·화나요·슬퍼요 (5종 이모지 + 수치)
- 같은 언론사 다른 기사 추천 리스트

**타임톡 (댓글)**:
- "타임톡" 섹션 헤더 + 댓글 수 + 기사 게시 후 경과 시간 타이머
- 댓글 입력창 + 정렬(최신순 기본) + 감정 이모지 필터
- "세이프봇이 작동 중입니다" 안내
- 댓글 구조: 닉네임 + 시간 + 본문 + 답글 쓰기 + 더보기(···)
- "V타임톡에서 다른 의견들 확인하기" CTA
- "타임톡 더보기" CTA

**기사 하단 추가 콘텐츠**:
- "관심 있을 만한 기사" — 개인화 추천 기사 카드 리스트 (썸네일+제목+출처)
- 추천 뉴스 피드 카드
- 실시간 트렌드 (순위형 5건 + 펼쳐보기)
- 루프(숏폼) 추천 가로 캐러셀
- 언론사 랭킹 뉴스 (많이 본 뉴스 / 탐독한 뉴스 탭 전환 + 순위 리스트)

**하단 바 (인앱 브라우저)**: Daum 로고 + ← → 네비게이션 + 검색 + 공유 + 더보기

### 3-2. 채널뷰

> **구현**: 인앱 브라우저 (웹뷰, v.daum.net)
> **진입**: 홈 피드·콘텐츠탭 등에서 채널 게시글 카드 탭
> **참조 이미지**: `screenshots/channel_view.png`

채널은 일반 사용자·크리에이터가 언론사처럼 글을 게시할 수 있는 준 공식 콘텐츠 플랫폼. 채널 게시글 전용 상세 화면으로, 뉴스 기사 상세(§3-1)와는 별도의 페이지.

**상단 바**: ← 뒤로가기 + 채널 아이콘·채널명 + 공유·더보기(···) 아이콘

**채널 프로필 영역**: 채널 아이콘 + 채널명 + 구독자 수 + "구독" 버튼

**게시글 본문 영역**:
- 제목 (대형 텍스트)
- 작성자명 · 작성일시 · 조회수
- 본문 텍스트 + 인라인 이미지 + 날짜 스탬프 등 리치 콘텐츠
- 인용문 블록 (배경색 구분)

**게시글 하단**:
- 저작권 안내 (Copyright 표시)
- 채널 프로필 카드 (아이콘 + 채널명 + 구독 버튼) — 본문 종료 후 재노출
- 좋아요(하트+수) + 댓글(말풍선+수) + 공유(종이비행기) 인터랙션 바

**댓글 영역**:
- "댓글" 섹션 헤더
- 댓글 리스트 (닉네임 + 시간 + 본문 + 답글)

**하단 추천 콘텐츠**:
- "당신을 위한 추천 콘텐츠" — 추천 채널 게시글/기사 카드 리스트
- "많은 관심을 받고 있는" — 인기 콘텐츠 카드 (대형 썸네일 + 제목 오버레이)
- 관련 콘텐츠 카드 추가 노출
- "정보 및 유의" 안내 영역

**하단 바 (인앱 브라우저)**: Daum 로고 + ← → 네비게이션 + 검색 + 공유 + 더보기

**뉴스 기사 상세와의 차이점**:

| 항목 | 뉴스 기사 상세 (§3-1) | 채널뷰 (§3-2) |
|------|---------------------|--------------|
| 콘텐츠 생산자 | 제휴 언론사 (CP) | 일반 사용자·크리에이터 |
| 상단 헤더 | 언론사 로고 + 언론사명 + 구독 | 채널 아이콘 + 채널명 + 공유·더보기 |
| 프로필 영역 | 언론사 헤더 (상단 1회) | 채널 프로필 (상단 + 본문 하단 재노출) |
| 본문 스타일 | 뉴스 기사 형태 (기자명·조회수·유틸리티 아이콘) | 블로그형 리치 콘텐츠 (인용문·이미지 중심) |
| 댓글 시스템 | 타임톡 (경과 시간 타이머 + 감정 이모지 필터 + 세이프봇) | 일반 댓글 |
| 감정 반응 | 5종 이모지 (추천해요·좋아요·감동이에요·화나요·슬퍼요) | 좋아요(하트) 중심 |
| 하단 추천 | 실시간 트렌드 + 추천뉴스 + 루프 + 언론사 랭킹 | 개인화 추천 콘텐츠 + 인기 콘텐츠 |

> **참조 이미지**: `screenshots/search_result.png`

- 하단: "최근 검색어 끄기" · "전체삭제" · "닫기"

**검색 결과 화면** *(구현: 인앱 브라우저, m.search.daum.net)*:
- 검색바: Daum 로고 + 검색어 + 삭제(×) + 음성검색(마이크)
- 결과 탭 바 (가로 스크롤): 쿼리 의도에 따라 탭 구성·순서가 동적으로 변경됨. 예) 뉴스성 쿼리 → 통합·뉴스·통합웹·동영상·쇼핑·책·이미지, 장소성 쿼리 → 통합·장소·통합웹·쇼핑·뉴스·동영상·이미지. 우측 끝에 설정(필터) 아이콘.
- 실시간 트렌드 배너 (접기/펼치기)
- 관련 검색어 칩 (가로 스크롤, 펼치기/접기)

**통합 검색 결과에 노출될 수 있는 섹션 유형**:

쿼리 의도에 따라 아래 섹션이 선택적으로 조합·순서가 변경되어 노출됨.

| 섹션 | 구성 | 비고 |
|------|------|------|
| 바로가기 | 특정 서비스·사이트로의 직접 링크 카드 | 명확한 네비게이션 의도 쿼리에서 노출 |
| 지하철역 정보 | 노선 탭(신분당·경강 등) + 실시간 도착 정보 + 시간표·노선도·길찾기·지도 버튼 + 시설정보 | 역명 쿼리에서 노출 |
| 파워링크 | "관련광고" 표시 + "AD"/"ⓘ" 마크. 광고주 URL + 제목 + 설명 + 썸네일. SA 광고 영역 | 광고 입찰이 있는 쿼리에서 노출 |
| 장소 | 카카오맵 연동. 지도 + 장소 리스트(이름·주소·별점·리뷰·전화·길찾기). 정확도순/거리순 정렬 | 장소성 쿼리에서 노출 |
| 뉴스 | 정확도순/최신순 정렬 + 뉴스검색 설정. 출처(언론사 아이콘+이름) + 제목(키워드 하이라이트) + 요약 + 시간 + 썸네일. "뉴스 더보기" | 뉴스성 쿼리에서 상위 노출 |
| 카페글 | Daum 카페 게시글 검색 결과. 카페명 + 제목 + 본문 요약 + 날짜. "카페글 검색 더보기" | — |
| 블로그 | 블로그 게시글 검색 결과. 출처 URL + 제목 + 본문 요약 + 이미지 + 날짜. "블로그 검색 더보기" | — |
| 웹문서 | 전체/블로그/카페/웹문서 필터 탭. 출처 URL + 제목 + 요약 + 날짜. "같은 출처 내 관련문서" 그룹핑. "웹문서 검색 더보기" | 대부분의 쿼리에서 노출 |
| 이미지 | 이미지 그리드 + "이미지 더보기" | — |
| 동영상 | 2×2 썸네일 그리드. 제목 + 플랫폼(Youtube 등) + 업로드 시간 + 채널명 | — |
| 다음 루프 | 숏폼 영상 가로 스크롤 캐러셀. 썸네일 + 제목 + 채널명 | 동영상 섹션 하단에 노출 |
