# M:AI 탭 PRD

> Daum 앱 하단 5번째 탭. 사용자의 AI 어시스턴트 진입점이자 개인 활동 허브.

## 목적과 배경

M:AI 탭은 두 가지 역할을 동시에 수행한다.

1. **AI 어시스턴트 진입점** — 사용자가 관심 키워드 기반 뉴스를 요약·플레이백으로 받아본다.
2. **개인 활동 허브** — 히스토리·구독·커뮤니티 활동을 한 곳에서 본다.

기존 마이 탭이 단순한 "내 정보 모음"이었다면, M:AI 탭은 **소비할 콘텐츠를 능동적으로 제안**한다.

## 상단 헤더 {#mai-top-bar}

화면 맨 위. 좌측 `M:AI` 워드마크, 우측 3개 아이콘 (알림·설정·검색).

- 알림 아이콘 → `/mai-notification` 로 이동
- 설정 아이콘 → `/mai-settings` 로 이동
- 검색 아이콘 → `/search` 로 이동
- 스크롤 시 sticky 유지 (현재 구현은 일반 헤더)

## 프로필 카드 {#mai-profile-header}

48×48 아바타 + 닉네임 + 이메일. 우측 chevron으로 프로필 상세 진입을 암시.

- 아바타: dicebear notionists 시드 기반 자동 생성
- 닉네임 폰트: 15px bold, gray-900
- 이메일 폰트: 12px, gray-500
- 클릭 시 동작: TBD (현재 미연결)

## 진입점 3카드 {#mai-gateway-row}

가로 3분할 grid. 각 카드는 `(라벨 / 카테고리·카운트)` 구조.

| 카드 | 라우트 | 카운트 의미 |
|---|---|---|
| 히스토리 | `/mai-history/recent` | 새 활동 수 |
| 구독 | `/mai-subscribe/channels` | 새 글 수 |
| 커뮤니티 | `/mai-sub/community` | 새 댓글 수 |

카운트는 빨간색(daum-red) 강조. 0이면 숨기는 게 아니라 0으로 표시.

## 서브탭 바 {#mai-subtab-bar}

상단 sticky로 고정되는 가로 서브탭. `새소식 / 이슈노트 / AI 탐구` 3개.

- 활성 탭은 굵은 검정 + 하단 인디케이터
- URL: `/mai/news`, `/mai/issue`, `/mai/explore`
- AI 탐구는 별도 ExploreProvider로 감싸 mainOverlay·fullOverlay 활용

## 새소식 피드 {#mai-news-feed}

`/mai/news` 기본 뷰. 필터 칩 + 뉴스 카드 리스트.

### 필터 칩
- 전체 / 내 주변 / 해시태그 (#이란전쟁, #주가, #유가 등)
- 가로 스크롤 가능. 현재 활성은 검정 pill, 나머지는 화이트 outline.

### 뉴스 카드
- 언론사·시각·실시간 LIVE 도트(있을 시)
- AI 키워드 매칭 배지 (`📊 이 키워드 이번 주 보도량 평소 대비 ×4.6`)
- 헤드라인 + 본문 요약 2~3줄 + 썸네일
- 하단 액션: `원문 보기 / 저장 / 공유 / ⋯`

## 이슈노트 피드 {#mai-issue-feed}

`/mai/issue` 뷰. 뉴스보다 한 단계 위의 이슈 단위 요약.

- 한 이슈당 카드 1개. 카드에 관련 기사 3~5건 묶음.
- 카드 진입 시 `MaiSubPage` 로 이동, 이슈 상세 노트 + 타임라인.
- 새소식과 다르게 **편집된 요약체**가 우선. 사실 나열 아닌 맥락 설명.

## AI 탐구 피드 {#mai-explore-feed}

`/mai/explore` 뷰. 사용자가 직접 AI에게 탐구 주제를 던지는 패널.

- 상단: 진행 중인 탐구 카드 (mainOverlay)
- 중단: 추천 탐구 주제 칩 (`이번 주 뉴스 한 줄 요약 / 내 관심 키워드 보고서` 등)
- 하단: 과거 아카이브
- "탐구 시작" 액션 → `CreateExploreSheet` 풀스크린 오버레이
